# API 开发者文档

## 调用限制

### 频率限制

API 调用有以下频率限制：

| 等级 | 每分钟请求数 | 每日请求数 |
|------|-------------|-----------|
| 免费版 | 60 | 1,000 |
| 专业版 | 600 | 50,000 |
| 企业版 | 6,000 | 500,000 |

### 配额管理

#### 查看当前配额

```http
GET /api/v1/quota
Authorization: Bearer YOUR_API_KEY
```

响应示例：
```json
{
  "minute_remaining": 45,
  "daily_remaining": 9800,
  "reset_at": "2024-01-15T00:00:00Z"
}
```

#### 配额计算规则

- 配额按自然时间周期重置
- 超出限制返回 429 状态码
- 建议实现指数退避重试

### 申请提升配额

1. 登录开发者控制台
2. 进入"API 管理" -> "配额提升"
3. 填写业务场景和使用预期
4. 提交审核（1-3 工作日）

### 错误码

| 状态码 | 含义 | 处理建议 |
|--------|------|----------|
| 429 | 请求过于频繁 | 降低请求频率，实现重试 |
| 403 | 配额已用完 | 等待重置或申请提升 |
| 401 | 认证失败 | 检查 API Key |
