# 消息队列服务文档

## 开通服务

### 快速开通

1. 登录云控制台
2. 进入"中间件" -> "消息队列"
3. 点击"立即开通"
4. 选择地域和规格
5. 完成支付（如付费版）

### 支持的协议

- HTTP / HTTPS（RESTful API）
- AMQP 0-9-1
- MQTT
- WebSocket

### 地域选择

| 地域 | 可用区 | 协议支持 |
|------|--------|----------|
| 华北-北京 | 可用区A/B/C | 全部 |
| 华东-上海 | 可用区A/B | 全部 |
| 华南-广州 | 可用区A/B | HTTP, AMQP |

## 死信队列（DLQ）

### 什么是死信队列

死信队列（Dead Letter Queue, DLQ）用于存放无法正常消费的消息。

### 触发条件

消息进入 DLQ 的条件：
1. 消费失败超过最大重试次数
2. 消息超过存活时间（TTL）
3. 消息被显式拒绝

### 配置示例

```json
{
  "queue_name": "order_processor",
  "dead_letter_queue": "order_processor_dlq",
  "max_retry": 3,
  "message_ttl": 86400000
}
```

### 处理 DLQ 消息

1. 登录控制台，进入 DLQ
2. 查看消息内容和失败原因
3. 修复问题后重新投递
4. 或手动消费处理
