# 高可用架构文档

## 心跳检测机制

### 默认配置

高可用模式（High Availability, HA）使用心跳机制检测节点状态：

- **默认心跳间隔**: 3 秒
- **心跳超时时间**: 9 秒（3 × 间隔）
- **选举超时时间**: 10 秒

### 主备切换

当主节点连续 3 次心跳失败后，系统自动触发主备切换：

1. 备用节点检测到主节点不可达
2. 等待选举超时后自动升级为主节点
3. 接管 VIP（虚拟 IP）和服务
4. 原主节点恢复后自动转为备用

### 配置参数

```yaml
ha:
  heartbeat_interval: 3s      # 心跳间隔
  failover_timeout: 9s        # 故障转移超时
  election_timeout: 10s       # 选举超时
  max_retry: 3               # 最大重试次数
```

### 监控建议

建议配置以下监控告警：
- 心跳失败率
- 主备切换次数
- VIP 漂移频率
